Als 10x-Entwickler ist der größte Produktivitätskiller, dem wir gegenüberstehen, der Aufwand beim Einrichten unserer Entwicklungsumgebung. Es ist ein Prozess, bei dem Dinge ohne Vorwarnung schiefgehen können.

Ich suchte nach einer Lösung für dieses Problem, als ich eines Tages bei ihrem Developers Club Tour Meetup in Neu-Delhi auf Daytona stieß. Aufgeregt sah ich ihre Creator Quest auf [Quira](https://quira.sh/quests/creator/details?questId=23).

## Was ist Daytona?

[Daytona](https://www.daytona.io/) ist ein selbst gehosteter und sicherer **Open-Source** Entwicklungsumgebungs-Manager.

![Daytona GitHub](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/6x6q69ztvir2vwo7yyux.png)

Es ist ein einfaches, unkompliziertes **CLI-Tool**, das die zusätzlichen Schritte wie das Herunterladen von Bibliotheken, Bauen, Aktualisieren, Kompilieren und Testen aus dem Entwicklungsprozess entfernt.

## Die App erstellen

Für die Quest habe ich ein Template zum Verbinden einer React-App mit Coinbase Wallet erstellt, unter Verwendung des Coinbase Wallet SDK.

![Coinbase Connect](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/shm1uaimxgzjj6dwwejo.png)

## React mit Coinbase SDK mit Daytona verwenden

Daytona zu installieren ist ein Kinderspiel, sogar unter Windows.

```bash
(curl -sf -L https://download.daytona.io/daytona/install.sh | sudo bash) && daytona server -y && daytona
```

Um es lokal auszuführen:

```bash
daytona create https://github.com/daytonaio/sample-react-coinbase-connect
```

Anwendung starten:

```bash
npm install
npm run dev
```

[GitHub](https://github.com/daytonaio/sample-react-coinbase-connect) (Gib ihm einen ⭐)
[Live Demo](https://coinbase-connect.vercel.app/)
