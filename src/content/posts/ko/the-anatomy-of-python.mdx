Python은 제 첫 번째 프로그래밍 언어였습니다. 단순함과 다재다능함으로 알려진 인기 있는 객체 지향 언어입니다. 인터프리터 언어로서 Python은 전체 코드를 한 번에 기계어로 컴파일하는 대신 인터프리터를 사용하여 코드를 한 줄씩 실행합니다.

이 글에서는 소스 코드 작성부터 실행까지 Python의 내부 작동 방식을 탐구합니다. 이러한 단계를 이해하면 개발자가 프로그래밍 언어로서 Python의 효율성과 힘을 이해하는 데 도움이 됩니다.

## 알아야 할 기본 용어

Python의 작동 방식에 들어가기 전에 몇 가지 기본 용어를 이해합시다:

### **코드 에디터**

소스 코드가 작성되는 환경입니다. Python 프로그램의 시작점으로, 개발자가 사람이 읽을 수 있는 형식으로 명령을 작성합니다.

### **소스 코드**

코드 에디터에서 작성되어 `.py` 파일로 저장된 사람이 읽을 수 있는 명령입니다. 이 파일에는 Python 컴파일러가 처리할 코드가 포함되어 있습니다.

### **컴파일 단계**

이 단계에서 Python 컴파일러는 소스 코드를 바이트 코드로 변환하며, 도중에 구문 오류를 확인합니다. 오류가 발견되지 않으면 바이트 코드를 포함하는 `.pyc` 파일이 생성됩니다.

### **Python 가상 머신 (PVM)**

PVM은 Python의 핵심 런타임 엔진입니다. 바이트 코드를 해석하고 CPU가 실행할 수 있는 기계 코드로 변환합니다. PVM은 Python 인터프리터가 있는 모든 머신에서 Python 코드를 실행할 수 있도록 보장하여 Python을 매우 이식 가능하게 만듭니다.

### **실행 중인 프로그램**

기계 코드가 생성되면 CPU가 이를 실행하고, 프로그램은 원래 Python 스크립트에서 지정한 작업과 계산을 수행합니다.

## Python의 실행 프로세스

Python 코드는 컴퓨터의 CPU에 의해 실행되기 전에 여러 단계를 거칩니다. Python 코드가 실행 가능한 코드로 변환되는 방법의 단계별 설명:

![코드 에디터](https://cdn.hashnode.com/res/hashnode/image/upload/v1715955234708/e46fbbb2-e325-4197-ae40-7511d6bfebb9.png)

1. ### 소스 코드 작성

   ![](https://live.staticflickr.com/4836/44661017890_eb4f288ffe_b.jpg)

   Python 프로그램의 첫 번째 단계는 코드 에디터에서 소스 코드를 작성하는 것입니다. 이는 Python의 구문 규칙에 따라 작성된 사람이 읽을 수 있는 코드입니다. 작성된 소스 코드는 `.py` 파일로 저장됩니다.

2. ### 바이트 코드로 컴파일

   ![](https://prepinstadotcom.s3.ap-south-1.amazonaws.com/wp-content/uploads/2020/07/Pythonbyte.webp)

   .py 파일이 실행되면 Python 컴파일러가 먼저 소스 코드를 읽습니다. 이 단계에서 컴파일러는 구문 오류를 확인합니다. 오류가 발견되지 않으면 컴파일러는 소스 코드를 바이트 코드로 변환하여 .pyc 파일을 생성합니다. 기계 코드와 달리 바이트 코드는 CPU에서 직접 실행할 수 없지만 중간 표현 역할을 합니다.

3. ### **Python 가상 머신 (PVM)에 의한 해석**

   그런 다음 바이트 코드는 Python 가상 머신(PVM)에 공급됩니다. PVM은 바이트 코드를 읽고 기계 실행 가능한 코드로 변환하는 인터프리터입니다. 이 변환은 한 줄씩 수행되며, PVM은 바이트 코드의 각 줄을 순차적으로 해석하고 실행합니다.

4. ### CPU에 의한 실행

   마지막으로 PVM에서 생성된 기계 코드는 CPU에 의해 실행됩니다. 이진 코드만 이해하는 CPU는 원래 Python 프로그램에서 지정한 작업과 계산을 수행하여 원하는 출력을 생성합니다.

## 요약

Python의 내부 작동 방식에는 소스 코드가 바이트 코드로 컴파일되고, PVM에 의해 해석되며, 최종적으로 CPU에 의해 기계 코드로 실행되는 다단계 프로세스가 포함됩니다.
