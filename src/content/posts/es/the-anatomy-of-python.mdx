Python fue mi primer lenguaje de programación. Es un lenguaje orientado a objetos popular conocido por su simplicidad y versatilidad. Como lenguaje interpretado, Python ejecuta código línea por línea usando un intérprete, en lugar de compilar todo el código a lenguaje máquina de una vez.

En este artículo, exploraremos el funcionamiento interno de Python, desde la escritura del código fuente hasta su ejecución. Entender estas etapas ayuda a los desarrolladores a apreciar la eficiencia y el poder de Python como lenguaje de programación.

## Términos Básicos a Conocer

Antes de sumergirnos en el funcionamiento de Python, primero entendamos algunos términos básicos:

### **Editor de Código**

Este es el entorno donde se escribe el código fuente. Es el punto de partida del programa Python, donde los desarrolladores escriben instrucciones en un formato legible por humanos.

### **Código Fuente**

Las instrucciones legibles por humanos escritas en el editor de código y guardadas como un archivo `.py`. Este archivo contiene el código que el compilador de Python procesará.

### **Etapa de Compilación**

En esta etapa, el compilador de Python traduce el código fuente a bytecode, verificando errores de sintaxis en el camino. Si no se encuentran errores, se genera un archivo `.pyc` que contiene el bytecode.

### **Máquina Virtual de Python (PVM)**

La PVM es el motor de ejecución central de Python. Interpreta el bytecode y lo traduce a código máquina que la CPU puede ejecutar. La PVM asegura que el código Python pueda ejecutarse en cualquier máquina con un intérprete de Python, haciendo Python altamente portable.

### **Programa en Ejecución**

Una vez que se genera el código máquina, la CPU lo ejecuta, y el programa realiza las tareas y cálculos especificados por el script Python original.

## Proceso de Ejecución de Python

El código Python pasa por varias etapas antes de ser ejecutado por la CPU de la computadora. Aquí hay un desglose paso a paso de cómo el código Python se convierte en código ejecutable:

![Editor de Código](https://cdn.hashnode.com/res/hashnode/image/upload/v1715955234708/e46fbbb2-e325-4197-ae40-7511d6bfebb9.png)

1. ### Escribiendo el Código Fuente

   ![](https://live.staticflickr.com/4836/44661017890_eb4f288ffe_b.jpg)

   El primer paso en cualquier programa Python es escribir el código fuente en un editor de código. Este es código legible por humanos, escrito de acuerdo con las reglas de sintaxis de Python. Una vez escrito, este código fuente se guarda como un archivo `.py`.

2. ### Compilación a Bytecode

   ![](https://prepinstadotcom.s3.ap-south-1.amazonaws.com/wp-content/uploads/2020/07/Pythonbyte.webp)

   Cuando se ejecuta el archivo .py, el compilador de Python primero lee el código fuente. Durante esta etapa, el compilador verifica errores de sintaxis. Si no se encuentran errores, el compilador convierte el código fuente en bytecode, resultando en un archivo .pyc. A diferencia del código máquina, el bytecode no es directamente ejecutable por la CPU pero sirve como una representación intermedia.

3. ### **Interpretación por la Máquina Virtual de Python (PVM)**

   El bytecode entonces se alimenta a la Máquina Virtual de Python (PVM). La PVM es un intérprete que lee el bytecode y lo traduce a código ejecutable por la máquina. Esta traducción ocurre línea por línea, lo que significa que la PVM interpreta y ejecuta cada línea de bytecode secuencialmente.

4. ### Ejecución por la CPU

   Finalmente, el código máquina producido por la PVM es ejecutado por la CPU. La CPU, que solo entiende código binario, realiza las tareas y cálculos especificados en el programa Python original, produciendo la salida deseada.

## Resumen

![](https://www.devopsschool.com/blog/wp-content/uploads/2020/08/python-virtual-machine.png)

El funcionamiento interno de Python involucra un proceso de múltiples etapas donde el código fuente se compila en bytecode, es interpretado por la PVM, y finalmente se ejecuta como código máquina por la CPU.
