Como desarrolladores 10x, el mayor asesino de productividad que enfrentamos es la molestia de configurar nuestro entorno de desarrollo. Es un proceso donde las cosas pueden salir mal sin previo aviso, lo que lleva a la frustración.

Estaba buscando una solución a este problema cuando un día me encontré con Daytona en su meetup del Developers Club Tour en Nueva Delhi. Emocionado, vi su Creator Quest en [Quira](https://quira.sh/quests/creator/details?questId=23).

Decidí intentar el desafío **023: Prueba Daytona** y en este blog, compartiré mi experiencia con Daytona.

## ¿Qué es Daytona?

[Daytona](https://www.daytona.io/) es un gestor de entornos de desarrollo **de código abierto**, auto-hospedado y seguro.

![Daytona GitHub](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/6x6q69ztvir2vwo7yyux.png)

Es una **herramienta CLI** simple, sin complicaciones, que elimina los pasos extra como descargar librerías, construir, actualizar, compilar y probar del proceso de desarrollo y te permite empezar a programar con **un solo comando**.

## Aprendiendo Daytona

Para tener una idea básica sobre Daytona, primero abrí su repositorio de [GitHub](https://github.com/daytonaio/daytona) y leí el `README.md`. Me impresionó el desarrollo activo y las 13.5k estrellas. Después de eso, revisé su [documentación](https://www.daytona.io/docs/) y [blog](https://www.daytona.io/dotfiles).

Luego elegí un tema para construir para el quest de los temas proporcionados por Daytona.

## Construyendo la App

Para el quest, construí una plantilla para conectarse a una app React con Coinbase Wallet, usando Coinbase Wallet SDK.

![Coinbase Connect](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/shm1uaimxgzjj6dwwejo.png)

Configuré un proyecto React + Vite, utilicé TypeScript, leí rápidamente la documentación del SDK de Coinbase. En una noche, tenía un MVP listo. Lo estilicé con shadcn/ui y TailwindCSS y lo desplegué en Vercel.

Aunque está enfocado en proyectos Web3 que involucran conectar Coinbase, esto también puede usarse como boilerplate de shadcn/ui jaja.

## Usando React con Coinbase SDK con Daytona

Esta debería ser la sección más completa del blog pero debido a la naturaleza ridículamente simple de Daytona, no hay mucho que hacer.

Primero que todo, instalar Daytona es muy fácil, incluso en Windows. Es una instalación de un solo comando y listo.

```powershell
$architecture = if ($env:PROCESSOR_ARCHITECTURE -eq "AMD64") { "amd64" } else { "arm64" }
md -Force "$Env:APPDATA\bin\daytona"; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.SecurityProtocolType]'Tls,Tls11,Tls12';
Invoke-WebRequest -URI "https://download.daytona.io/daytona/v0.50/daytona-windows-$architecture.exe" -OutFile "$Env:APPDATA\bin\daytona\daytona.exe";
$env:Path += ";" + $Env:APPDATA + "\bin\daytona"; [Environment]::SetEnvironmentVariable("Path", $env:Path, [System.EnvironmentVariableTarget]::User);
daytona serve;
```

Si estás en Linux o MacOS, puedes instalar Daytona con este comando:

```bash
(curl -sf -L https://download.daytona.io/daytona/install.sh | sudo bash) && daytona server -y && daytona
```

Para ejecutarlo localmente, asegúrate de tener Docker instalado y funcionando en tu máquina. Luego ejecuta el siguiente comando para crear tu workspace:

```bash
daytona create https://github.com/daytonaio/sample-react-coinbase-connect
```

![Terminal](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/q2e9nwy9dcvjxa36qi58.png)

Iniciar la Aplicación:

```bash
npm install
npm run dev

# Construyendo para Producción
npm run build
npm run preview
```
