# Tailwind v3 → v4 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the held-back Tailwind upgrade by migrating from v3 (JS config + `tailwind.config.ts` + `tailwindcss-animate`) to v4 (CSS-first config + `@tailwindcss/postcss` + `tw-animate-css`) without changing visual output.

**Architecture:** v4 reads its config from CSS via `@theme` and `@import`. The custom shadcn `plugin()` and `theme.extend` blocks in `tailwind.config.ts` get ported into `src/styles/globals.css` as `@theme inline` mappings plus `:root` CSS variables. The MDX prose styles already live in `mdx.css` and use plain `@apply` against utility classes (`bg-muted`, `text-foreground`, etc.), so they keep working unchanged once the same utilities exist in v4.

**Tech Stack:** Tailwind CSS 4.x, `@tailwindcss/postcss`, `@tailwindcss/typography` 0.5.x (v4-compatible), `tw-animate-css` (replacement for `tailwindcss-animate`), PostCSS, Next.js 16 Turbopack.

## Task 3: Rewrite `globals.css` with v4 CSS-first config

**Files:**
- Modify: `src/styles/globals.css`

This task ports everything from `tailwind.config.ts` (CSS variables, color mappings, custom font families, custom screens, custom keyframes/animations, `addBase` from the shadcn plugin) into CSS using v4's `@import`, `@plugin`, `@theme`, and `@layer` directives.

- [ ] **Step 1: Replace globals.css with the v4-equivalent stylesheet**

Replace the entire contents of `src/styles/globals.css` with:

```css
@import "tailwindcss";
@import "tw-animate-css";

@plugin "@tailwindcss/typography";

/* Custom screens (matches v3 config: xs/sm overrides) */
@theme {
  --breakpoint-xs: 380px;
  --breakpoint-sm: 500px;

  --font-sans: var(--font-inter-tight), ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, sans-serif;
  --font-serif: var(--font-instrument-serif), ui-serif, Georgia, serif;
  --font-heading: var(--font-instrument-serif), ui-serif, Georgia, serif;
  --font-mono: var(--font-jetbrains-mono), ui-monospace, SFMono-Regular, Menlo, monospace;

  --animate-accordion-down: accordion-down 0.2s ease-out;
  --animate-accordion-up: accordion-up 0.2s ease-out;
}

/* shadcn CSS vars — channel form for hsl(var(--*)) usage in components */
:root {
  --background: 0 0% 0%;
  --foreground: 0 0% 100%;
  --card: 0 0% 0%;
  --card-foreground: 0 0% 100%;
  --popover: 0 0% 0%;
  --popover-foreground: 0 0% 100%;
  --primary: 0 0% 100%;
  --primary-foreground: 0 0% 0%;
  --secondary: 0 0% 10%;
  --secondary-foreground: 0 0% 100%;
  --muted: 0 0% 15%;
  --muted-foreground: 0 0% 65%;
  --accent: 0 0% 15%;
  --accent-foreground: 0 0% 100%;
  --destructive: 0 0% 30%;
  --destructive-foreground: 0 0% 100%;
  --border: 0 0% 20%;
  --input: 0 0% 15%;
  --ring: 0 0% 100%;
  --radius: 0.5rem;
}

/* Map shadcn vars into Tailwind v4 utilities. `inline` keeps references live so
   any future dark-mode override on :root flows into bg-*/text-* automatically. */
@theme inline {
  --color-background: hsl(var(--background));
  --color-foreground: hsl(var(--foreground));
  --color-card: hsl(var(--card));
  --color-card-foreground: hsl(var(--card-foreground));
  --color-popover: hsl(var(--popover));
  --color-popover-foreground: hsl(var(--popover-foreground));
  --color-primary: hsl(var(--primary));
  --color-primary-foreground: hsl(var(--primary-foreground));
  --color-secondary: hsl(var(--secondary));
  --color-secondary-foreground: hsl(var(--secondary-foreground));
  --color-muted: hsl(var(--muted));
  --color-muted-foreground: hsl(var(--muted-foreground));
  --color-accent: hsl(var(--accent));
  --color-accent-foreground: hsl(var(--accent-foreground));
  --color-destructive: hsl(var(--destructive));
  --color-destructive-foreground: hsl(var(--destructive-foreground));
  --color-border: hsl(var(--border));
  --color-input: hsl(var(--input));
  --color-ring: hsl(var(--ring));

  --radius-lg: var(--radius);
  --radius-md: calc(var(--radius) - 2px);
  --radius-sm: calc(var(--radius) - 4px);
}

@keyframes accordion-down {
  from {
    height: 0;
  }
  to {
    height: var(--radix-accordion-content-height);
  }
}
@keyframes accordion-up {
  from {
    height: var(--radix-accordion-content-height);
  }
  to {
    height: 0;
  }
}

/* Base layer — replaces the addBase calls from the shadcn plugin in v3 */
@layer base {
  * {
    @apply border-border;
  }
  body {
    @apply bg-background text-foreground;
  }
}

/* Lenis smooth scroll styles */
html.lenis,
html.lenis body {
  height: auto;
}

.lenis.lenis-smooth {
  scroll-behavior: auto !important;
}

.lenis.lenis-smooth [data-lenis-prevent] {
  overscroll-behavior: contain;
}

.lenis.lenis-stopped {
  overflow: hidden;
}

@layer components {
  .el-focus-styles {
    @apply ring-offset-background focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2;
  }

  ::selection {
    @apply bg-white/20 text-white;
  }
}

/* Typography plugin: customize prose colors to match shadcn palette
   (replaces the typography.extend block from tailwind.config.ts) */
.prose {
  --tw-prose-body: hsl(var(--foreground));
  --tw-prose-headings: hsl(var(--foreground));
  --tw-prose-links: hsl(var(--foreground));
  --tw-prose-bold: hsl(var(--foreground));
  --tw-prose-counters: hsl(var(--muted-foreground));
  --tw-prose-bullets: hsl(var(--muted-foreground));
  --tw-prose-hr: hsl(var(--border));
  --tw-prose-quotes: hsl(var(--muted-foreground));
  --tw-prose-quote-borders: hsl(var(--foreground));
  --tw-prose-captions: hsl(var(--muted-foreground));
  --tw-prose-code: hsl(var(--foreground));
  --tw-prose-pre-code: hsl(var(--foreground));
  --tw-prose-pre-bg: hsl(var(--muted));
  --tw-prose-th-borders: hsl(var(--border));
  --tw-prose-td-borders: hsl(var(--border));

  max-width: 100ch;
}

.prose :where(h1, h2, h3, h4, h5, h6) {
  font-family: var(--font-heading);
}

.prose :where(a:hover) {
  color: hsl(var(--muted-foreground));
}
```

## Notes for the implementer

- **Do not** restore class-based dark mode. The site uses CSS-variable theming only (`--background` is hardcoded to `0 0% 0%`); no `dark:` variant strategy was configured in v3 and none is needed in v4.
- **Do not** delete `src/styles/mdx.css` — its `@apply` calls reference utilities that v4 still produces (`bg-muted`, `text-foreground`, `border-border`, etc.) once `globals.css` defines the matching `--color-*` tokens. The build will tell you immediately if any are missing.
- The `@import "tw-animate-css"` line provides drop-in replacements for everything `tailwindcss-animate` shipped (the radix-ui accordion / sheet / dialog / dropdown packages depend on `data-state` animations from that library). If a Radix component animates wrong, that's the line to investigate first.
- Pre-bundled `@tailwindcss/typography` v0.5.x works with v4 via the `@plugin` directive — no separate package change needed.
